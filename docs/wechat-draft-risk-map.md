# 微信公众号草稿写入 Risk Map

## 发布安全边界

- 系统只能保存草稿。
- 系统只能生成预览。
- 系统不得自动发布。
- 系统不得自动群发。
- 系统不得确认任何最终发送弹窗。
- 最终发布必须人工操作。

## 风险地图

| risk | trigger | impact | mitigation | shouldStop | requiresHumanAction |
| --- | --- | --- | --- | --- | --- |
| 登录态过期 | 页面跳回登录页、接口提示未登录、二维码重新出现 | 无法继续写入草稿，后续操作可能落在错误页面 | 立即停止，提示人工重新扫码登录；不保存 cookie/token | true | true |
| 公众号后台页面改版 | 图文入口、草稿箱入口、编辑器或按钮文案变化 | 自动化选择器或人工 SOP 可能失效，存在误点风险 | 停止并重新人工 review 页面结构，更新 SOP 后再继续 | true | true |
| HTML 粘贴后样式丢失 | 编辑器过滤 inline style、段落结构丢失、标题层级变化 | 草稿展示质量下降，可能无法达到公众号排版要求 | 生成预览并人工核对；必要时回退 HTML 排版阶段修正 | false | true |
| 封面上传失败 | 上传组件报错、文件不可读、格式不被接受、网络失败 | 草稿无法达到发布前最低完整度 | 停止草稿写入，检查 `cover.json` 的 `imagePath` 和文件格式 | true | true |
| 标题过长 | 公众号后台提示标题超限或无法保存 | 草稿保存失败或标题被截断 | 停止，人工缩短标题并重新进入审核/排版链路 | true | true |
| 摘要为空或过长 | 摘要字段为空、超出后台限制或保存时报错 | 预览质量下降，可能阻止保存或影响转发卡片 | 停止并由人工确认摘要文本；必要时从 article meta 重新生成 | true | true |
| 页面误点“发布” | 光标或选择器落到发布按钮，按钮区域与保存草稿接近 | 可能进入最终发布链路 | `forbidAutoPublish` 拦截 action label；页面出现发布入口时停止，不点击 | true | true |
| 页面误点“群发” | 草稿页或发布页出现群发按钮，误识别为保存动作 | 可能触发面向用户的群发链路 | `forbidAutoPublish` 拦截 action label；发现群发按钮时停止并人工确认 | true | true |
| 弹窗出现“确认发送” | 点击错误按钮或后台弹出最终发送确认 | 可能导致最终发送确认被误点 | 任何包含“确认发送”的弹窗都必须停止；不得确认 | true | true |
| 草稿保存失败 | 后台提示保存失败、网络错误、字段校验失败 | 草稿不存在或内容不完整 | 停止并记录失败原因；人工判断是否重试保存草稿 | true | true |
| 图片比例不符合要求 | 封面不是目标比例、裁切异常、后台提示尺寸问题 | 封面显示变形或无法保存 | 停止并回到封面生成/裁切阶段；确认 900x383 要求 | true | true |
| 公众号编辑器过滤部分 HTML | 粘贴后 `section`、`blockquote`、`hr`、inline style 或图片样式被过滤 | 实际草稿与 `outputs/wechat.html` 不一致 | 生成预览，人工检查差异；必要时调整 HTML 到公众号兼容子集 | false | true |
| 网络超时 | 页面加载、图片上传、保存草稿或预览生成超时 | 操作状态不确定，可能重复提交或漏保存 | 停止并人工刷新确认状态；不要连续点击保存或发送相关按钮 | true | true |
| 二维码登录需要人工扫码 | 后台展示登录二维码或要求管理员确认 | 无法自动进入后台，且涉及账号安全 | 等待人工扫码；不绕过登录，不保存敏感凭据 | true | true |

## 风险处理原则

1. 所有发布、群发、最终发送确认相关风险都必须 `shouldStop=true`。
2. 登录、保存、上传、标题摘要校验失败时必须停止并等待人工处理。
3. 排版显示差异可以先生成预览，但必须人工确认，不得进入发布或群发。
4. 后续 9B 即使进入真实后台测试，也只能保存草稿、生成预览、等待人工确认。
