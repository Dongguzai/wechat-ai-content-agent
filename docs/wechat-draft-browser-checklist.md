# 微信公众号草稿浏览器测试 Checklist

## 发布安全边界

- 系统只能保存草稿。
- 系统只能生成预览。
- 系统不得自动发布。
- 系统不得自动群发。
- 系统不得确认任何最终发送弹窗。
- 最终发布必须人工操作。

## 运行前检查

- [ ] 文章审核通过：`outputs/article-review.json` 的 `passed=true`。
- [ ] 封面审核通过：`outputs/cover-review.json` 的 `passed=true`。
- [ ] HTML 安全通过：`outputs/wechat-layout.json` 的 `allowedNextStage=true`，且 `compatibleWithWechat=true`。
- [ ] `outputs/wechat.html` 存在且非空。
- [ ] `outputs/cover.json` 的 `imagePath` 存在，且封面文件可读取。
- [ ] mock draft dry-run 通过：`outputs/wechat-draft-result.json` 的 `mode="mock"` 且 `status="draft_saved"`。
- [ ] mock draft dry-run 明确 `allowedNextStage=false`。
- [ ] `forbidAutoPublish` 已能拦截“发布”“群发”“确认发送”“立即发送”。
- [ ] 用户确认进入真实后台测试。
- [ ] 用户准备人工扫码登录。
- [ ] 用户确认不会自动发布、不会自动群发，最终发布必须人工操作。

## 运行中检查

- [ ] 当前页面是否在公众号后台。
- [ ] 是否进入新建图文页面。
- [ ] 标题是否填写成功。
- [ ] 摘要是否填写成功。
- [ ] 正文是否粘贴成功。
- [ ] 正文 HTML 的标题、段落、引用、分割线和图片是否仍可见。
- [ ] 封面是否上传成功。
- [ ] 封面裁切、比例和显示位置是否正常。
- [ ] 是否只点击“保存草稿”。
- [ ] 是否只生成预览。
- [ ] 是否没有点击“发布”。
- [ ] 是否没有点击“群发”。
- [ ] 是否没有点击“确认发送”。
- [ ] 是否没有点击“立即发送”。
- [ ] 出现未知弹窗、发送确认弹窗、登录失效或页面改版时是否立即停止。

## 运行后检查

- [ ] 草稿是否存在。
- [ ] 预览是否正常。
- [ ] 移动端显示是否正常。
- [ ] 封面是否正常。
- [ ] 标题是否正常。
- [ ] 摘要是否正常。
- [ ] 是否等待人工确认。
- [ ] 是否没有发布。
- [ ] 是否没有群发。
- [ ] 是否没有确认任何最终发送弹窗。
- [ ] 是否未保存 cookie、token、账号密码、二维码或登录态文件到仓库。

## 停止条件

只要命中以下任一条件，必须停止：

- 登录态过期。
- 公众号后台页面改版，无法确认按钮语义。
- HTML 粘贴后内容明显丢失或错乱。
- 封面上传失败。
- 标题或摘要不符合后台限制。
- 出现“发布”“群发”“确认发送”“立即发送”相关按钮或弹窗。
- 草稿保存失败。
- 网络超时。
- 用户未完成扫码登录。
- 用户未确认继续测试。
